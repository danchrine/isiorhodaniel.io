SCOPE & RULES OF ENGAGEMENT

METHODOLOGY

1. RECONNAISSANCE

  • Subdomain enumeration (Amass, Sublist3r)
  • DNS reconnaissance (DNSdumpster)
  • Technology fingerprinting (Wappalyzer)

2. VULNERABILITY ANALYSIS

  • Automated scanning (Burp Suite Pro, Nikto)
  • Manual testing (OWASP ZAP, custom scripts)
  • Dependency checking (Retire.js, Snyk)

3. EXPLOITATION

  • SQLi verification (SQLMap, manual payloads)
  • Session hijacking (Burp Sequencer)
  • Business logic testing (manual)

4. POST-EXPLOITATION

  • Data extraction proofs (limited, non-disruptive)
  • Privilege escalation attempts
  • Lateral movement simulation

KEY FINDINGS

1. SQL INJECTION (CWE-89) — CVSS 8.8 (HIGH)

The /search endpoint was vulnerable to time-based blind SQL injection, allowing unauthorized database access.

PROOF OF CONCEPT 
sqlmap -u "https://toriloacademy.com/search?q=term" --risk=3 --level=5 --dump

Successful extraction of partial user table (redacted in report)

REMEDIATION: Implement parameterized queries with prepared statements. Apply input validation using allow-lists. Consider WAF rules for immediate mitigation.

2. OUTDATED COMPONENTS (CWE-1104) — CVSS 6.5 (MEDIUM)

Server headers revealed outdated jQuery (v2.1.4) with known XSS vulnerabilities (CVE-2020-11022).

REMEDIATION: Update all dependencies to latest secure versions. Implement dependency management with Snyk or Dependabot for automated alerts.

3. INFORMATION DISCLOSURE (CWE-200) — CVSS 3.1 (LOW)

Verbose error messages exposed internal paths and stack traces to unauthenticated users.

REMEDIATION: Configure custom error pages. Disable debug mode in production. Implement generic error messages.

  2. Short-term (1 week): Update all outdated components and dependencies

  4. Follow-up: Schedule retest after remediation (included in SOW)
