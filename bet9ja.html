Bet9ja Penetration Test

Black-Box Assessment of a High-Traffic Betting Platform

Overview

This penetration test was conducted against Bet9ja.com, one of Nigeriaâ€™s largest betting platforms. The goal was to evaluate the security posture using black-box testing techniques simulating an external attacker.

  • Assessment Type: Black-Box Penetration Test
  • Scope: bet9ja.com, subdomains, public infrastructure
  • Tools Used: Nmap, Masscan, Nikto, SQLmap, Hydra, Medusa, Metasploit, WPScan, Dirb, WAFW00F, SpiderFoot
  • Duration: 2 weeks

Methodology

1. Reconnaissance

WHOIS, DNS enumeration, Sublist3r, and SpiderFoot were used to collect intelligence on Bet9jaâ€™s assets.

2. Scanning & Enumeration

Nmap and Masscan were used for port/service detection. Nikto and WPScan identified web server misconfigurations and potential plugin vulnerabilities.

3. Exploitation

SQLmap was tested against exposed endpoints. Hydra and Medusa were used for credential brute-forcing against SSH/FTP where applicable.

4. Post-Exploitation

Metasploit was leveraged for privilege escalation on test environments. Collected proof-of-concept evidence while maintaining ethical boundaries.

Key Findings

1. Subdomain Exposure

Over 50 subdomains discovered, some hosting outdated services with weak SSL configurations.

sublist3r -d bet9ja.com

2. Web Server Misconfigurations

Nikto scan revealed outdated Apache modules and directory listings enabled on staging environments.

nikto -h https://staging.bet9ja.com

3. Potential SQL Injection Vectors

SQLmap tests flagged potential injection points, though exploitation was restricted to prevent data compromise.

sqlmap -u "https://bet9ja.com/page.php?id=5" --risk=3 --level=5 --dbs

4. Brute Force Attempts

Hydra testing on login portals confirmed the need for account lockout mechanisms.

hydra -l admin -P passwords.txt bet9ja.com https-post-form "/login:username=^USER^&password=^PASS^:F=incorrect"

Recommendations

  • Enforce account lockout policies to prevent brute-force attacks.
  • Update and patch outdated web servers and plugins.
  • Harden SSL/TLS configurations and disable weak ciphers.
  • Regular security monitoring and penetration testing.
