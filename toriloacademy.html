Executive Summary

A black-box assessment of Torilo Academy’s public-facing assets was performed with a focus on reconnaissance, service enumeration, and non-destructive web security testing. Several medium- and low-severity issues were identified, including outdated components and missing security headers. No destructive exploitation was conducted; testing adhered to the agreed rules of engagement.

1. Scope & Rules of Engagement

  • Type: Black-box (unauthenticated, external)
  • In-scope: Public domains/subdomains identified via OSINT and DNS enumeration
  • Out-of-scope: DoS, credential stuffing, data destruction, social engineering
  • Data handling: No sensitive data retained. Only metadata and proofs of concept captured.
  • Impact limits: Non-intrusive; exploitation limited to safe proof-of-concept checks.

2. Methodology

  1. Recon & OSINT: WHOIS, certificate transparency logs, search operators.
  2. DNS & Subdomains: Passive/active enumeration (e.g., Sublist3r); verification via DNS lookups.
  3. Port/Service Scanning: Nmap (TCP/UDP top ports) and service fingerprinting.
  4. Web Footprinting: HTTP headers, tech stack ID, WAF detection.
  5. Vulnerability Mapping: Nikto baseline checks, WPScan (if WordPress detected), directory discovery (Dirb).
  6. Validation: Manual verification; no destructive payloads executed.
Example commands (redacted/representative):

nmap -sV -sC -Pn -T4 target.tld
nikto -h https://target.tld
wpscan --url https://blog.target.tld --enumerate vp,vt,u
dirb https://target.tld /usr/share/wordlists/dirb/common.txt

3. Findings & Risk Ratings

FindingSeverityEvidence (summary)Remediation
Missing Security Headers (HSTS, CSP, X-Frame-Options) Medium Response headers lacked HSTS/CSP; increases MITM/XSS framing risk. Enable HSTS, implement a restrictive CSP, add X-Frame-Options/SameSite cookies.
Outdated CMS/Plugin Detected High WPScan indicated outdated plugin versions with known CVEs. Patch CMS and plugins; enable routine update policy and plugin allowlist.
Directory Listing Enabled Low Certain paths returned auto-index pages exposing files. Disable autoindex at the web server or via .htaccess; restrict direct object access.
Weak TLS Config (incomplete hardening) Medium Deprecated ciphers/protocols present according to header/scan checks. Disable legacy ciphers; prefer TLS 1.2+/modern ciphers; enable OCSP stapling.
Information Leakage via Error Pages Low Verbose stack traces/server banners observed during edge cases. Use custom error pages; suppress stack traces; sanitize server signature.

5. Recommendations

  • Implement security headers (HSTS/CSP/Frame-Options) and cookie hardening (HttpOnly/SameSite/secure).
  • Patch CMS & plugins; adopt monthly update and vulnerability monitoring cadence.
  • Harden TLS per modern baselines; remove deprecated protocols/ciphers.
  • Disable directory indexing; enforce least-privilege file permissions.
  • Introduce a WAF with virtual patching for rapid mitigation of emerging CVEs.

6. Conclusion

The environment shows generally good exposure control with opportunities to improve baseline hardening and patch governance. Addressing the findings above will reduce attack surface and improve resilience against common web threats.
